# Skills Backlog

This backlog tracks the next implementation steps for the skills knowledge harness.
Order is prioritized for impact and dependency safety.

## Phase 1: Foundation

- [ ] Add examples in each skill file using `datavolley` Python snippets.
- [ ] Add a shared glossary file and link it from all skill docs.
- [ ] Add traceability blocks in each skill file with source URL and retrieval date.
- [ ] Define a status tag format (`draft`, `validated`, `needs-source`) and apply it.

## Phase 2: Data Contract Hardening

- [ ] Build a canonical metric dictionary with formulas and denominators.

- [ ] Document leakage checks and split strategies.
- [ ] Add calibration checks and threshold policy examples.
- [ ] Add uncertainty reporting guidance for segment-level predictions.

- [ ] Add direction-normalization conventions for court charts.
